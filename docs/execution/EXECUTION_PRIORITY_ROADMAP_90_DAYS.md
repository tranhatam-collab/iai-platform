# EXECUTION_PRIORITY_ROADMAP_90_DAYS.md
## IAI — 90 Day Real Build Plan
## Version 1.0

---

# 0. NGUYÊN TẮC

- Không build nhiều → build đúng lõi
- Không làm đẹp → làm chạy thật
- Không mở rộng → khóa 1 use case
- Không delay → ship mỗi tuần

---

# 1. MỤC TIÊU 90 NGÀY

Phải đạt được:

- 1 sản phẩm dùng được mỗi tuần (Life OS + Flow đơn giản)
- 100–300 user thật
- 20–50 case “trước–sau”
- 1 flow chạy mỗi ngày/user

---

# 2. TUẦN 1–2 (LOCK CORE)

## Build
- DB schema (D1)
- Run + Step engine
- Queue system
- Artifact storage
- Basic API

## Deliverable
- 1 flow chạy được end-to-end

---

# 3. TUẦN 3–4 (MVP USER)

## Build
- LifeCode check-in (7 câu)
- Dashboard 4 chỉ số
- Run trigger đơn giản

## Deliverable
- User đăng nhập → check-in → thấy kết quả

---

# 4. TUẦN 5–6 (FLOW + ACTION)

## Build
- 5 flow mẫu:
  - Focus reset
  - Sleep check
  - Work plan
  - Weekly review
  - Risk alert

## Deliverable
- User dùng flow mỗi tuần

---

# 5. TUẦN 7–8 (AGENT + INSIGHT)

## Build
- AI summary tuần
- Pattern detection (burnout)
- Suggest 3 actions

## Deliverable
- Insight thực tế

---

# 6. TUẦN 9–10 (PROOF + TRUST)

## Build
- Proof ID cho mỗi run
- Verify endpoint
- Dash hiển thị proof

## Deliverable
- User tin hệ thống

---

# 7. TUẦN 11–12 (COMMUNITY + LOOP)

## Build
- nhóm 30 người
- sharing case
- weekly call

## Deliverable
- retention bắt đầu

---

# 8. KPI

- WAU ≥ 100
- retention ≥ 25%
- flow/run ≥ 3/tuần/user
- case thật ≥ 20

---

# 9. FINAL DIRECTIVE

90 ngày = sống hay chết

Không hoàn thành → không có hệ
